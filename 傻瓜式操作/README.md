# DeepSeek-OCR macOS 傻瓜式操作版

一键安装、一键使用的 DeepSeek-OCR 工具包，适用于 macOS 系统。

## 特点

✅ **零配置** - 自动安装所有依赖
✅ **通用性** - 复制到任何位置都能直接使用
✅ **macOS 优化** - CPU 模式，无需 GPU
✅ **中文友好** - 全中文交互界面
✅ **批量处理** - 自动处理所有图片

## 快速开始

### 首次使用（约 10-20 分钟）

1. 打开终端（Terminal）
2. 将 `install.sh` 拖到终端窗口
3. 按回车键
4. 等待安装完成

### 日常使用

1. 把图片放到 `test_images/` 文件夹
2. 将 `ocr.sh` 拖到终端窗口，按回车
3. 选择识别模式（输入 1-5）
4. 在 `ocr_output/` 查看结果

## 识别模式

1. **文档转 Markdown** (推荐) - 保留格式和结构
2. **普通 OCR** - 提取所有文字
3. **无布局 OCR** - 纯文本提取
4. **图表解析** - 解析图表内容
5. **详细描述** - 图片详细描述

## 系统要求

- macOS 10.15 或更高版本
- 至少 15GB 可用磁盘空间
- 网络连接（首次下载模型）

## 文件说明

- `install.sh` - 安装脚本（仅首次运行）
- `ocr.sh` - OCR 识别脚本
- `fix_cuda.py` - CUDA 兼容性修复（自动调用）
- `convert_mmd.py` - 格式转换工具（自动调用）
- `test_images/` - 放置待识别图片
- `ocr_output/` - 识别结果输出目录
- `使用顺序.txt` - 详细使用说明

## 常见问题

**Q: 第一次运行很慢？**
A: 正常，需要下载约 10GB 的模型文件，只有第一次慢

**Q: 识别一张图片要多久？**
A: 约 3-5 分钟（CPU 模式）

**Q: 支持什么图片格式？**
A: PNG, JPG, JPEG, BMP, WEBP

**Q: 可以移动到其他位置吗？**
A: 可以！把整个文件夹复制到任何位置都能用

## 技术说明

- 基于 DeepSeek-OCR 模型
- 使用 PyTorch CPU 模式
- Conda 环境隔离
- 自动修复 CUDA 兼容性问题
